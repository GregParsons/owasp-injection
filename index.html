A05:2025 Injection

What is Injection?

Injection is an attacker's attempt to send data to an application in a way that will change the meaning of commands being sent to an interpreter.

An interpreter can be a browser, application, database or command line that executes user input via form input fields, query parameters, headers, cookies or file uploads.

Types of Injection

There are many types of injection vulnerability, including:

  • SQL Injection - modifying a SQL command to retrieve unintended or unauthorised data from, or cause damage to a database
  • Cross-Site Scripting (XSS) - inserting malicious JavaScript that runs when user visits a site
  • Command Injection - mimicing commands that the web application would normally send to the operating system
  • XPath Injection - similar to SQL injection but for XML databases
  • LDAP Injection - execution of arbitrary commands such as granting permissions to unauthorized queries from a single sign-on system
  • Mail Command Injection - exploit email servers and services that build IMAP and SMTP statements to send spam messages

Example: XSS

A common feature of websites and applications is allowing user created data to be submitted and displayed to other users. If this data is not properly sanitised, a malicious user can inject HTML or JavaScript.

Example: SQL Injection

A SQL query is contructed using input from the user, in this case, the user's email address.

SELECT credit_card_number FROM credit_cards WHERE user = {email};

If the user enters a malicious input, they can alter the SQL query:

Resulting in a modified query:

SELECT credit_card_number FROM credit_cards WHERE user = gregory.parsons@otago.ac.nz OR 1 = 1;

This will potentially return every record in the credit_cards table.

Preventing Injections

The most important rule is do not trust the user. Assume they can and will do everything to attack and break your application. Some of the methods to protect against injection vulnerabilites include:

  • Input Validation - If you are expect the input to follow a certain pattern, enforce this, where possible.
  • Query Parameterization - Don't use raw SQL queries. Instead, use query parameterization to prepare, validate and sanitise your query.
  • Output Encoding - Encode the output by escaping characters.
